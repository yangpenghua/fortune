# 修改项目端口 — 技术方案

## 背景与目标

项目当前使用 Next.js 默认端口 3000 启动开发服务器。为避免端口冲突或符合特定的端口使用规范，需要将开发服务器端口修改为 9527。

## 方案设计

Next.js 修改端口有两种常用方式：

### 方案一：修改 package.json scripts（推荐）
在 `dev` 命令中添加 `-p 9527` 参数，简单直接，无需额外配置文件。

### 方案二：使用 .env 文件
创建 `.env` 文件，设置 `PORT=9527`。

本次采用方案一，修改 package.json 即可。

## 部署注意事项

- 开发环境：修改后执行 `npm run dev` 将在 http://localhost:9527 启动
- 生产环境：不受影响，生产部署使用 `next start`，端口通常由部署平台配置
