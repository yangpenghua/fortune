## 背景与目标

### 背景
根据用户反馈，AI 算命应用中的"星座运势"选项存在不能点击的问题，导致用户无法使用该功能进行测算。经过初步代码审查，发现星座运势功能在代码中已定义，但可能存在交互或逻辑问题。

### 目标
1. 排查并修复星座运势选项不能点击的问题
2. 确保星座运势功能完整可用，与其他测算风格体验一致
3. 验证星座运势从选择到结果展示的完整流程

## 方案设计

### 现有架构分析

**前端流程**：
```
用户选择测算风格 → 点击开始测算 → 调用 API → 展示结果
```

**涉及文件**：
- `src/components/form/FortuneForm.tsx` - 测算表单组件
- `src/app/page.tsx` - 首页，处理表单提交
- `src/types/index.ts` - 类型定义
- `src/lib/doubao/prompts.ts` - AI 提示词配置
- `src/lib/doubao/client.ts` - AI 客户端

### 问题排查方向

1. **表单组件交互检查**：验证星座运势按钮的 `onClick` 事件是否正确绑定
2. **状态管理检查**：验证 `formData.style` 状态是否正确更新
3. **类型定义检查**：确认 `FortuneStyle` 类型包含 `'constellation'`
4. **禁用状态检查**：确认按钮未被意外禁用

### 修复方案

根据代码审查结果，现有实现理论上应该正常工作。如需修复，按以下优先级：

## 接口定义

**已有 API**：`POST /api/fortune`

请求字段：
```typescript
{
  name: string           // 姓名（2-10个汉字）
  gender?: 'male' | 'female' | 'other'  // 性别（可选）
  birthDate?: string     // 出生日期（可选，YYYY-MM-DD格式）
  style?: 'traditional' | 'constellation' | 'tarot'  // 测算风格
}
```

响应字段：
```typescript
{
  success: boolean
  data?: FortuneData
  error?: string
}
```
